O EmDash expõe uma API REST em `/_emdash/api/` para gerenciamento de conteúdo, uploads de mídia e operações de esquema.

## Autenticação

As requisições à API exigem autenticação via token Bearer:

```http
Authorization: Bearer <token>
```

Gere tokens através da interface administrativa ou programaticamente.

## Formato da Resposta

Todas as respostas seguem um formato consistente:

```json
// Success
{
  "success": true,
  "data": { ... }
}

// Erro
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Mensagem legível",
    "details": { ... }
  }
}
```

## Endpoints de Conteúdo

### Listar Conteúdo

```http
GET /_emdash/api/content/:collection
```

#### Parâmetros

| Parâmetro    | Tipo      | Descrição                                 |
| ------------ | --------- | ----------------------------------------- |
| `collection` | `string`  | Slug da coleção (caminho)                 |
| `cursor`     | `string`  | Cursor de paginação (consulta)            |
| `limit`      | `number`  | Itens por página (consulta, padrão: 50)   |
| `status`     | `string`  | Filtrar por status (consulta)             |
| `orderBy`    | `string`  | Campo para ordenar (consulta)             |
| `order`      | `string`  | Direção da ordenação: `asc` ou `desc` (consulta) |

#### Resposta

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "01HXK5MZSN...",
        "type": "posts",
        "slug": "hello-world",
        "data": { "title": "Ola mundo", ... },
        "status": "published",
        "createdAt": "2025-01-24T12:00:00Z",
        "updatedAt": "2025-01-24T12:00:00Z"
      }
    ],
    "nextCursor": "eyJpZCI6..."
  }
}
```

### Obter Conteúdo

```http
GET /_emdash/api/content/:collection/:id
```

#### Resposta

```json
{
  "success": true,
  "data": {
    "item": {
      "id": "01HXK5MZSN...",
      "type": "posts",
      "slug": "hello-world",
      "data": { "title": "Ola mundo", ... },
      "status": "published",
      "createdAt": "2025-01-24T12:00:00Z",
      "updatedAt": "2025-01-24T12:00:00Z"
    }
  }
}
```

### Criar Conteúdo

```http
POST /_emdash/api/content/:collection
Content-Type: application/json
```

#### Corpo da Requisição

```json
{
  "data": {
    "title": "New Post",
    "content": [...]
  },
  "slug": "new-post",
  "status": "draft"
}
```

#### Resposta

```json
{
  "success": true,
  "data": {
    "item": { ... }
  }
}
```

### Atualizar Conteúdo

```http
PUT /_emdash/api/content/:collection/:id
Content-Type: application/json
```

#### Corpo da Requisição

```json
{
	"data": {
		"title": "Updated Title"
	},
	"status": "published"
}
```

### Excluir Conteúdo

```http
DELETE /_emdash/api/content/:collection/:id
```

#### Resposta

```json
{
	"success": true,
	"data": {
		"success": true
	}
}
```

## Endpoints de Mídia

### Listar Mídia

```http
GET /_emdash/api/media
```

#### Parâmetros

| Parâmetro  | Tipo      | Descrição                          |
| ---------- | --------- | ---------------------------------- |
| `cursor`   | `string`  | Cursor de paginação                |
| `limit`    | `number`  | Itens por página (padrão: 20)      |
| `mimeType` | `string`  | Filtrar por prefixo de tipo MIME   |

#### Resposta

```json
{
	"success": true,
	"data": {
		"items": [
			{
				"id": "01HXK5MZSN...",
				"filename": "photo.jpg",
				"mimeType": "image/jpeg",
				"size": 102400,
				"width": 1920,
				"height": 1080,
				"url": "https://cdn.example.com/photo.jpg",
				"createdAt": "2025-01-24T12:00:00Z"
			}
		],
		"nextCursor": "eyJpZCI6..."
	}
}
```

### Obter Mídia

```http
GET /_emdash/api/media/:id
```

### Criar Mídia

```http
POST /_emdash/api/media
Content-Type: application/json
```

#### Corpo da Requisição

```json
{
	"filename": "photo.jpg",
	"mimeType": "image/jpeg",
	"size": 102400,
	"width": 1920,
	"height": 1080,
	"storageKey": "uploads/photo.jpg"
}
```

<Aside>
	Este endpoint registra os metadados da mídia após o upload. Use o endpoint de upload ou URLs assinadas para
	fazer o upload do arquivo real.
</Aside>

### Atualizar Mídia

```http
PUT /_emdash/api/media/:id
Content-Type: application/json
```

#### Corpo da Requisição

```json
{
	"alt": "Photo description",
	"caption": "Photo caption"
}
```

### Excluir Mídia

```http
DELETE /_emdash/api/media/:id
```

### Obter Arquivo de Mídia

```http
GET /_emdash/api/media/file/:key
```

Serve o conteúdo real do arquivo. Apenas para armazenamento local.

#### Parâmetros

| Parâmetro | Tipo      | Descrição                                 |
| --------- | --------- | ----------------------------------------- |

```json
  }
```

```http
GET /_emdash/api/revisions/:revisionId
```

```http
POST /_emdash/api/revisions/:revisionId/restore
```

Restaura o conteúdo para o estado desta revisão e cria uma nova revisão.

## Endpoints de Esquema

### Listar Coleções

```http
GET /_emdash/api/schema/collections
```

#### Resposta

```json
{
	"success": true,
	"data": {
		"items": [
			{
				"id": "01HXK5MZSN...",
				"slug": "posts",
				"label": "Posts",
				"labelSingular": "Post",
				"supports": ["drafts", "revisions", "preview"]
			}
		]
	}
}
```

### Obter Coleção

```http
GET /_emdash/api/schema/collections/:slug
```

#### Parâmetros

| Parâmetro        | Tipo       | Descrição                               |
| ---------------- | ---------- | --------------------------------------- |
| `includeFields`  | `boolean`  | Incluir definições de campos (consulta) |

### Criar Coleção

```http
POST /_emdash/api/schema/collections
Content-Type: application/json
```

#### Corpo da Requisição

```json
{
	"slug": "products",
	"label": "Products",
	"labelSingular": "Product",
	"description": "Product catalog",
	"supports": ["drafts", "revisions"]
}
```

### Atualizar Coleção

```http
PATCH /_emdash/api/schema/collections/:slug
Content-Type: application/json
```

### Excluir Coleção

```http
DELETE /_emdash/api/schema/collections/:slug
```

#### Parâmetros

| Parâmetro | Tipo       | Descrição                                           |
| --------- | ---------- | --------------------------------------------------- |
| `force`   | `boolean`  | Excluir mesmo se a coleção tiver conteúdo (consulta) |

### Listar Campos

```http
GET /_emdash/api/schema/collections/:slug/fields
```

### Criar Campo

```http
POST /_emdash/api/schema/collections/:slug/fields
Content-Type: application/json
```

#### Corpo da Requisição

```json
{
	"slug": "price",
	"label": "Price",
	"type": "number",
	"required": true,
	"validation": {
		"min": 0
	}
}
```

### Atualizar Campo

```http
PATCH /_emdash/api/schema/collections/:collectionSlug/fields/:fieldSlug
Content-Type: application/json
```

### Excluir Campo

```http
DELETE /_emdash/api/schema/collections/:collectionSlug/fields/:fieldSlug
```

### Reordenar Campos

```http
POST /_emdash/api/schema/collections/:slug/fields/reorder
Content-Type: application/json
```

#### Corpo da Requisição

```json
{
	"fieldSlugs": ["title", "content", "author", "publishedAt"]
}
```

## Códigos de Erro

| Código                     | Status HTTP | Descrição                          |
| -------------------------- | ----------- | ---------------------------------- |
| `NOT_FOUND`                | 404         | Recurso não encontrado             |
| `VALIDATION_ERROR`         | 400         | Dados de entrada inválidos         |
| `UNAUTHORIZED`             | 401         | Token ausente ou inválido          |
| `FORBIDDEN`                | 403         | Permissões insuficientes           |
| `CONTENT_LIST_ERROR`       | 500         | Falha ao listar conteúdo           |
| `CONTENT_CREATE_ERROR`     | 500         | Falha ao criar conteúdo            |
| `CONTENT_UPDATE_ERROR`     | 500         | Falha ao atualizar conteúdo        |
| `CONTENT_DELETE_ERROR`     | 500         | Falha ao excluir conteúdo          |
| `MEDIA_LIST_ERROR`         | 500         | Falha ao listar mídia              |
| `MEDIA_CREATE_ERROR`       | 500         | Falha ao criar mídia               |
| `SCHEMA_ERROR`             | 400         | Operação de esquema falhou         |
| `DUPLICATE_SLUG`           | 409         | Slug já existe                     |
| `RESERVED_SLUG`            | 400         | Slug é reservado                   |

## Endpoints de Busca

### Busca Global

```http
GET /_emdash/api/search?q=hello+world
```

#### Parâmetros

| Parâmetro     | Tipo      | Descrição                                      |
| ------------- | --------- | ---------------------------------------------- |
| `q`           | `string`  | Consulta de busca (obrigatório)                |
| `collections` | `string`  | Slugs de coleções separados por vírgula        |
| `status`      | `string`  | Filtrar por status (padrão: publicado)         |
| `limit`       | `number`  | Máximo de resultados (padrão: 20)              |
| `cursor`      | `string`  | Cursor de paginação                            |

#### Resposta

```json
{
  "results": [
    {
      "collection": "posts",
      "id": "01HXK5MZSN...",
      "slug": "hello-world",
      "title": "Ola mundo",
      "snippet": "...this is a <mark>hello</mark>
 <mark>world</mark>
 example...",
      "score": 0.95
    }
  ],
  "nextCursor": "eyJvZmZzZXQiOjIwfQ"
}
```

### Sugestões de Busca

```http
GET /_emdash/api/search/suggest?q=hel&limit=5
```

Retorna títulos com correspondência de prefixo para autocompletar.

### Reconstruir Índice de Busca

```http
POST /_emdash/api/search/rebuild
```

Reconstrói o índice FTS para todas ou coleções específicas.

### Estatísticas de Busca

```http
GET /_emdash/api/search/stats
```

Retorna contagens de documentos indexados por coleção.

## Endpoints de Autenticação

### Status de Configuração

```http
GET /_emdash/api/setup/status
```

Retorna se a configuração está completa e se existem usuários.

### Login com Chave de Acesso

```http
POST /_emdash/api/auth/passkey/options
```

Obter opções de autenticação WebAuthn.

```http
POST /_emdash/api/auth/passkey/verify
Content-Type: application/json

{
  "id": "credential-id",
  "rawId": "...",
  "response": {...},
  "type": "public-key"
}
```

Verificar chave de acesso e criar sessão.

### Link Mágico

```http
POST /_emdash/api/auth/magic-link/send
Content-Type: application/json

{
  "email": "user@example.com"
}
```

```http
GET /_emdash/api/auth/magic-link/verify?token=xxx
```

### Logout

```http
POST /_emdash/api/auth/logout
```

### Usuário Atual

```http
GET /_emdash/api/auth/me
```


## Limitação de Taxa

Os endpoints da API podem ter limitação de taxa com base na configuração de implantação. Quando limitados, as respostas incluem:

```http
HTTP/1.1 429 Too Many Requests
Retry-After: 60
```

## CORS

A API suporta CORS para requisições de navegador. Configure origens permitidas na sua implantação.
