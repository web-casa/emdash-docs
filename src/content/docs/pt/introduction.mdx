O EmDash é um **sistema de gerenciamento de conteúdo nativo do Astro**. Ele traz padrões familiares de CMS — coleções, taxonomias, menus, widgets e uma interface de administração polida — diretamente para o seu site Astro, com suporte completo a TypeScript e implantação portátil.

## O que é o EmDash

O EmDash é um CMS construído especificamente para [Astro](https://astro.build). Ele usa as Coleções de Conteúdo ao Vivo do Astro 6 para servir conteúdo em tempo de execução sem reconstruções. O conteúdo é armazenado em bancos de dados compatíveis com SQLite (D1, libSQL, SQLite local) e mídia em armazenamento compatível com S3 (R2, sistema de arquivos local).

**Características principais:**

- **Esquema primeiro no banco de dados** — Coleções e campos são definidos no banco de dados, não no código. Crie e modifique tipos de conteúdo a partir da interface de administração.
- **Coleções ao Vivo** — Alterações de conteúdo ficam disponíveis imediatamente. Nenhuma reconstrução estática é necessária.
- **Sistema de plugins** — Hooks inspirados no WordPress, extensões de armazenamento, configurações e interface de administração.
- **Portátil para a nuvem** — Executa no Cloudflare (Workers + D1 + R2), Node.js, SQLite local e qualquer armazenamento compatível com S3.

## O que o EmDash não é

- **Não é um CMS headless** — O EmDash é totalmente integrado ao Astro. Não é um serviço separado que você chama via API.
- **Não é compatível com WordPress** — Sem PHP, sem plugins do WordPress rodando diretamente. Mas o conteúdo e os conceitos do WordPress migram de forma limpa.
- **Não é um construtor de páginas** — O EmDash foca em conteúdo estruturado. Para construção visual de páginas, use componentes do Astro.

## Conceitos Principais

Antes de mergulhar, familiarize-se com estes conceitos-chave:

- **Coleções** — Tipos de conteúdo definidos no banco de dados (posts, páginas, produtos, etc.)
- **Campos** — As propriedades de uma coleção (título, conteúdo, preço, etc.)
- **Taxonomias** — Sistemas de classificação (categorias, tags, taxonomias personalizadas)
- **Menus** — Estruturas de navegação editáveis pelo administrador
- **Áreas de Widgets** — Regiões de conteúdo configuráveis para barras laterais e rodapés
- **Plugins** — Extensões que adicionam funcionalidade via hooks, armazenamento e interface
