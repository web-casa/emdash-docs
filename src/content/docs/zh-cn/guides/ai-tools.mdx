EmDash 内置了一个 [MCP 服务器](https://modelcontextprotocol.io)，让 AI 助手可以直接操作你站点里的内容。你可以让 Claude、ChatGPT 或其他工具起草文章、更新页面、管理媒体、搜索内容等，整个过程都可以通过自然语言完成。

## 启用 MCP 服务器

MCP 服务器默认是关闭的。请先在 Astro 配置中启用它：

```js title="astro.config.mjs"
emdash({
  mcp: true,
})
```

## 设置

您站点的 MCP 服务器 URL 是：

```
https://example.com/_emdash/api/mcp
```

请把 `example.com` 替换成你的域名。进行本地开发时，请使用 `http://localhost:4321/_emdash/api/mcp`。

<Tabs>
<TabItem label="Claude">

在 [claude.ai](https://claude.ai) 中添加的连接器在网页应用和 Claude Desktop 中均可使用。

<Steps>

1. 前往 [设置 > 连接器](https://claude.ai/settings/connectors)
2. 点击 **添加自定义连接器**
3. 输入您站点的 MCP 服务器 URL
4. 点击 **添加** —— 您的浏览器将打开，供您登录并批准访问
5. 开始新的对话，点击聊天输入框旁的 **+**，然后点击 **连接器**，并打开您站点的开关

</Steps>

对于团队和企业计划，所有者需要先从 [管理员设置 > 连接器](https://claude.ai/admin-settings/connectors) 添加连接器。然后成员从各自的设置中进行个人连接。

</TabItem>
<TabItem label="ChatGPT">

ChatGPT 在 Pro、Business 和 Enterprise 计划中支持 MCP 服务器。

<Steps>

1. 前往 **设置 > 应用与连接器 > 高级设置**，并启用 **开发者模式**
2. 前往 **设置 > 连接器 > 创建**
3. 输入名称、描述和您站点的 MCP 服务器 URL
4. 点击 **创建**
5. 在对话中，点击输入框附近的 **+**，然后点击 **更多**，并选择您的连接器

</Steps>

## 您可以做什么

连接后，您可以用自然语言要求 AI 助手执行以下任何操作。您不需要知道工具名称——只需描述您想要什么。

### 内容

-   **浏览内容** —— “给我看看最新的 10 篇博客文章” 或 “找到所有草稿页面”
-   **读取内容** —— “获取名为 ‘hello-world’ 的文章并总结它”
-   **创建内容** —— “写一篇关于我们夏季促销的新博客文章” 或 “为‘关于’部分创建一个草稿页面”
-   **编辑内容** —— “更新定价页面以提及新计划” 或 “修复 FAQ 文章中的拼写错误”
-   **发布和安排** —— “发布夏季促销文章” 或 “将公告安排在 6 月 1 日上午 9 点”
-   **比较版本** —— “给我看看主页自上次发布以来有什么变化”
-   **管理草稿** —— “丢弃关于页面的草稿更改” 或 “复制新闻稿模板”
-   **翻译** —— “欢迎文章的翻译有哪些？”（当 i18n 启用时）

### 媒体

-   **浏览媒体** —— “列出所有上传的图片” 或 “给我看看媒体库中的 PDF 文件”
-   **检查详情** —— “获取此媒体项的详细信息”
-   **更新元数据** —— “将主图上的替代文本设置为 ‘Mountain sunset’”
-   **删除文件** —— “删除旧的横幅图片”

### 搜索

-   **查找内容** —— “搜索提到 ‘accessibility’ 的文章” 或 “在所有集合中查找任何关于 TypeScript 的内容”

### 分类法

-   **浏览** —— “列出所有分类” 或 “给我看看标签”
-   **创建术语** —— “添加一个 ‘tutorials’ 标签” 或 “在 ‘Engineering’ 下创建一个 ‘Frontend’ 子分类”

### 菜单

-   **查看菜单** —— “给我看看主导航菜单” 或 “页脚菜单里有什么？”

### 模式（仅限管理员）

-   **检查** —— “存在哪些集合？” 或 “给我看看 posts 集合上的字段”
-   **创建集合** —— “创建一个新的 ‘testimonials’ 集合，包含 name 和 quote 字段”
-   **修改模式** —— “向 posts 添加一个 ‘featured’ 布尔字段”

<Aside type="caution">
	模式更改（创建/删除集合和字段）会修改您的数据库结构，需要管理员权限。如果您没有足够的访问权限，AI 会告知您。
</Aside>

### 修订

-   **查看历史** —— “显示此文章的修订历史”
-   **恢复** —— “将文章恢复到其之前的版本”

## 权限

您通过 AI 工具可以执行的操作取决于您在 EmDash 中的角色。AI 助手拥有与您在管理面板中相同的权限：

| 角色 | AI 可以做什么 |
| --- | --- |
| **管理员** | 所有操作，包括模式更改 |
| **编辑** | 所有内容、媒体、分类法和菜单。可以查看模式。 |
| **作者** | 自己的内容和媒体 |
| **投稿者** | 自己的内容（无发布权限）和媒体 |

如果您尝试执行没有权限的操作，AI 会告知您。

## 提示

-   **明确指定集合。** 如果您有多个集合，请说“创建一篇博客文章”而不是“创建一篇文章”。
-   **先询问模式。** 如果不确定某个集合有哪些字段，在创建或编辑内容之前，先问“posts 集合有哪些字段？”
-   **发布前进行审查。** 让 AI 创建草稿内容，在管理面板中审查，然后让 AI 发布它——或者您自己发布。
-   **使用比较功能进行审查。** 发布前，询问“比较这篇文章的实时版本和草稿版本”以确切了解将更改什么。
-   **富文本字段使用 Portable Text。** AI 可以为富文本字段编写内容，但复杂的格式化最好在管理编辑器中进行。
