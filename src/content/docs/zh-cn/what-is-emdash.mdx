EmDash 是一个面向 Astro 的原生 CMS，适合希望继续使用 Astro，同时又需要完整内容管理工作流的团队。它把管理面板、类型安全内容模型、Astro 集成以及更贴近应用代码的部署方式结合在了一起。

如果要用一句话概括：EmDash 是把 WordPress 里的集合、分类法、菜单、小部件和媒体管理这些熟悉概念，重新放到 Astro、TypeScript 和现代部署模型里实现的一套内容系统。

## EmDash 适合谁

- 已经在使用 Astro，想要真正 CMS 工作流的团队
- 重视类型安全和运行时内容更新的开发者
- 需要管理面板而不是直接改 Markdown 文件的编辑团队
- 想从 WordPress 迁移，但又不想失去熟悉内容模型的用户

## EmDash 的差异点

- **专为 Astro 构建**：不是在通用 CMS 上额外加 Astro 适配
- **单一部署模型**：内容系统和前端站点位于同一应用表面
- **类型安全内容访问**：更适合 TypeScript 和 Astro 项目协作
- **保留熟悉的内容管理方式**：分类法、菜单、小部件、预览等都属于核心能力
