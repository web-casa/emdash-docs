EmDash stellt eine REST-API unter `/_emdash/api/` für Inhaltsverwaltung, Medien-Uploads und Schema-Operationen bereit.

## Authentifizierung

API-Anfragen erfordern eine Authentifizierung über einen Bearer-Token:

```http
Authorization: Bearer <token>
```

Generieren Sie Token über die Admin-Oberfläche oder programmatisch.

## Antwortformat

Alle Antworten folgen einem einheitlichen Format:

```json
// Erfolg
{
  "success": true,
  "data": { ... }
}

// Fehler
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Menschenlesbare Nachricht",
    "details": { ... }
  }
}
```

## Inhalts-Endpunkte

### Inhalte auflisten

```http
GET /_emdash/api/content/:collection
```

#### Parameter

| Parameter    | Typ       | Beschreibung                             |
| ------------ | -------- | --------------------------------------- |
| `collection` | `string` | Collection-Slug (Pfad)                  |
| `cursor`     | `string` | Paginierungs-Cursor (Query)               |
| `limit`      | `number` | Elemente pro Seite (Query, Standard: 50)     |
| `status`     | `string` | Nach Status filtern (Query)                |
| `orderBy`    | `string` | Feld zum Sortieren (Query)                |
| `order`      | `string` | Sortierrichtung: `asc` oder `desc` (Query) |

#### Antwort

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "01HXK5MZSN...",
        "type": "posts",
        "slug": "hello-world",
        "data": { "title": "Hallo Welt", ... },
        "status": "published",
        "createdAt": "2025-01-24T12:00:00Z",
        "updatedAt": "2025-01-24T12:00:00Z"
      }
    ],
    "nextCursor": "eyJpZCI6..."
  }
}
```

### Inhalt abrufen

```http
GET /_emdash/api/content/:collection/:id
```

#### Antwort

```json
{
  "success": true,
  "data": {
    "item": {
      "id": "01HXK5MZSN...",
      "type": "posts",
      "slug": "hello-world",
      "data": { "title": "Hallo Welt", ... },
      "status": "published",
      "createdAt": "2025-01-24T12:00:00Z",
      "updatedAt": "2025-01-24T12:00:00Z"
    }
  }
}
```

### Inhalt erstellen

```http
POST /_emdash/api/content/:collection
Content-Type: application/json
```

#### Anfrage-Body

```json
{
  "data": {
    "title": "Neuer Beitrag",
    "content": [...]
  },
  "slug": "new-post",
  "status": "draft"
}
```

#### Antwort

```json
{
  "success": true,
  "data": {
    "item": { ... }
  }
}
```

### Inhalt aktualisieren

```http
PUT /_emdash/api/content/:collection/:id
Content-Type: application/json
```

#### Anfrage-Body

```json
{
	"data": {
			"title": "Aktualisierter Titel"
	},
	"status": "published"
}
```

### Inhalt löschen

```http
DELETE /_emdash/api/content/:collection/:id
```

#### Antwort

```json
{
	"success": true,
	"data": {
		"success": true
	}
}
```

## Medien-Endpunkte

### Medien auflisten

```http
GET /_emdash/api/media
```

#### Parameter

| Parameter  | Typ       | Beschreibung                  |
| ---------- | -------- | ---------------------------- |
| `cursor`   | `string` | Paginierungs-Cursor            |
| `limit`    | `number` | Elemente pro Seite (Standard: 20) |
| `mimeType` | `string` | Nach MIME-Typ-Präfix filtern   |

#### Antwort

```json
{
	"success": true,
	"data": {
		"items": [
			{
				"id": "01HXK5MZSN...",
				"filename": "photo.jpg",
				"mimeType": "image/jpeg",
				"size": 102400,
				"width": 1920,
				"height": 1080,
				"url": "https://cdn.example.com/photo.jpg",
				"createdAt": "2025-01-24T12:00:00Z"
			}
		],
		"nextCursor": "eyJpZCI6..."
	}
}
```

### Medium abrufen

```http
GET /_emdash/api/media/:id
```

### Medium erstellen

```http
POST /_emdash/api/media
Content-Type: application/json
```

#### Anfrage-Body

```json
{
	"filename": "photo.jpg",
	"mimeType": "image/jpeg",
	"size": 102400,
	"width": 1920,
	"height": 1080,
	"storageKey": "uploads/photo.jpg"
}
```

<Aside>
	Dieser Endpunkt erfasst Medien-Metadaten nach dem Upload. Verwenden Sie den Upload-Endpunkt oder signierte URLs, um
	die eigentliche Datei hochzuladen.
</Aside>

### Medium aktualisieren

```http
PUT /_emdash/api/media/:id
Content-Type: application/json
```

#### Anfrage-Body

```json
{
	"alt": "Bildbeschreibung",
	"caption": "Bildunterschrift"
}
```

### Medium löschen

```http
DELETE /_emdash/api/media/:id
```

### Mediendatei abrufen

```http
GET /_emdash/api/media/file/:key
```

Liefert den eigentlichen Dateiinhalt. Nur für lokalen Speicher.

## Revisions-Endpunkte

### Revisionen auflisten

```http
GET /_emdash/api/content/:collection/:entryId/revisions
```

#### Parameter

| Parameter | Typ       | Beschreibung                           |
| --------- | -------- | ------------------------------------- |
| `limit`   | `number` | Maximale zurückzugebende Revisionen (Standard: 50) |

#### Antwort

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "01HXK5MZSN...",
        "collection": "posts",
        "entryId": "01HXK5MZSN...",
        "data": { ... },
        "createdAt": "2025-01-24T12:00:00Z"
      }
    ],
    "total": 5
  }
}
```

### Revision abrufen

```http
GET /_emdash/api/revisions/:revisionId
```

### Revision wiederherstellen

```http
POST /_emdash/api/revisions/:revisionId/restore
```

Stellt den Inhalt auf den Zustand dieser Revision wieder her und erstellt eine neue Revision.

## Schema-Endpunkte

### Collections auflisten

```http
GET /_emdash/api/schema/collections
```

#### Antwort

```json
{
	"success": true,
	"data": {
		"items": [
			{
				"id": "01HXK5MZSN...",
				"slug": "posts",
				"label": "Beiträge",
				"labelSingular": "Beitrag",
				"supports": ["drafts", "revisions", "preview"]
			}
		]
	}
}
```

### Collection abrufen

```http
GET /_emdash/api/schema/collections/:slug
```

#### Parameter

| Parameter       | Typ        | Beschreibung                       |
| --------------- | --------- | --------------------------------- |
| `includeFields` | `boolean` | Felddefinitionen einbeziehen (Query) |

### Collection erstellen

```http
POST /_emdash/api/schema/collections
Content-Type: application/json
```

#### Anfrage-Body

```json
{
	"slug": "products",
	"label": "Products",
	"labelSingular": "Product",
	"description": "Product catalog",
	"supports": ["drafts", "revisions"]
}
```

### Collection aktualisieren

```http
PATCH /_emdash/api/schema/collections/:slug
Content-Type: application/json
```

### Collection löschen

```http
DELETE /_emdash/api/schema/collections/:slug
```

#### Parameter

| Parameter | Typ        | Beschreibung                                   |
| --------- | --------- | --------------------------------------------- |
| `force`   | `boolean` | Löschen auch wenn Collection Inhalte hat (Query) |

### Felder auflisten

```http
GET /_emdash/api/schema/collections/:slug/fields
```

### Feld erstellen

```http
POST /_emdash/api/schema/collections/:slug/fields
Content-Type: application/json
```

#### Anfrage-Body

```json
{
	"slug": "price",
	"label": "Price",
	"type": "number",
	"required": true,
	"validation": {
		"min": 0
	}
}
```

### Feld aktualisieren

```http
PATCH /_emdash/api/schema/collections/:collectionSlug/fields/:fieldSlug
Content-Type: application/json
```

### Feld löschen

```http
DELETE /_emdash/api/schema/collections/:collectionSlug/fields/:fieldSlug
```

### Felder neu anordnen

```http
POST /_emdash/api/schema/collections/:slug/fields/reorder
Content-Type: application/json
```

#### Anfrage-Body

```json
{
	"fieldSlugs": ["title", "content", "author", "publishedAt"]
}
```

## Fehlercodes

| Code                   | HTTP-Status | Beschreibung              |
| ---------------------- | ----------- | ------------------------ |
| `NOT_FOUND`            | 404         | Ressource nicht gefunden       |
| `VALIDATION_ERROR`     | 400         | Ungültige Eingabedaten       |
| `UNAUTHORIZED`         | 401         | Fehlender oder ungültiger Token |
| `FORBIDDEN`            | 403         | Unzureichende Berechtigungen |
| `CONTENT_LIST_ERROR`   | 500         | Inhaltsauflistung fehlgeschlagen   |
| `CONTENT_CREATE_ERROR` | 500         | Inhaltserstellung fehlgeschlagen |
| `CONTENT_UPDATE_ERROR` | 500         | Inhaltsaktualisierung fehlgeschlagen |
| `CONTENT_DELETE_ERROR` | 500         | Inhaltslöschung fehlgeschlagen |
| `MEDIA_LIST_ERROR`     | 500         | Medienauflistung fehlgeschlagen     |
| `MEDIA_CREATE_ERROR`   | 500         | Medienerstellung fehlgeschlagen   |
| `SCHEMA_ERROR`         | 400         | Schema-Operation fehlgeschlagen  |
| `DUPLICATE_SLUG`       | 409         | Slug existiert bereits      |
| `RESERVED_SLUG`        | 400         | Slug ist reserviert         |

## Such-Endpunkte

### Globale Suche

```http
GET /_emdash/api/search?q=hello+world
```

#### Parameter

| Parameter     | Typ       | Beschreibung                      |
| ------------- | -------- | -------------------------------- |
| `q`           | `string` | Suchanfrage (erforderlich)          |
| `collections` | `string` | Kommagetrennte Collection-Slugs |
| `status`      | `string` | Nach Status filtern (Standard: published) |
| `limit`       | `number` | Maximale Ergebnisse (Standard: 20)        |
| `cursor`      | `string` | Paginierungs-Cursor                |

#### Antwort

```json
{
  "results": [
    {
      "collection": "posts",
      "id": "01HXK5MZSN...",
      "slug": "hello-world",
      "title": "Hallo Welt",
      "snippet": "...this is a <mark>hello</mark>
 <mark>world</mark>
 example...",
      "score": 0.95
    }
  ],
  "nextCursor": "eyJvZmZzZXQiOjIwfQ"
}
```

### Suchvorschläge

```http
GET /_emdash/api/search/suggest?q=hel&limit=5
```

Gibt präfix-gematchte Titel für die Autovervollständigung zurück.

### Suchindex neu aufbauen

```http
POST /_emdash/api/search/rebuild
```

Baut den FTS-Index für alle oder bestimmte Collections neu auf.

### Suchstatistiken

```http
GET /_emdash/api/search/stats
```

Gibt die Anzahl der indizierten Dokumente pro Collection zurück.

## Authentifizierungs-Endpunkte

### Einrichtungsstatus

```http
GET /_emdash/api/setup/status
```

Gibt zurück, ob die Einrichtung abgeschlossen ist und ob Benutzer existieren.

### Passkey-Login

```http
POST /_emdash/api/auth/passkey/options
```

WebAuthn-Authentifizierungsoptionen abrufen.

```http
POST /_emdash/api/auth/passkey/verify
Content-Type: application/json

{
  "id": "credential-id",
  "rawId": "...",
  "response": {...},
  "type": "public-key"
}
```

Passkey verifizieren und Sitzung erstellen.

### Magic Link

```http
POST /_emdash/api/auth/magic-link/send
Content-Type: application/json

{
  "email": "user@example.com"
}
```

```http
GET /_emdash/api/auth/magic-link/verify?token=xxx
```

### Abmelden

```http
POST /_emdash/api/auth/logout
```

### Aktueller Benutzer

```http
GET /_emdash/api/auth/me
```

### Benutzer einladen

```http
POST /_emdash/api/auth/invite
Content-Type: application/json

{
  "email": "newuser@example.com",
  "role": 30
}
```

### Passkey-Verwaltung

```http
GET /_emdash/api/auth/passkey
```

Passkeys des Benutzers auflisten.

```http
POST /_emdash/api/auth/passkey/register/options
POST /_emdash/api/auth/passkey/register/verify
```

Neuen Passkey registrieren.

```http
PATCH /_emdash/api/auth/passkey/:id
Content-Type: application/json

{
  "name": "MacBook Pro"
}
```

Passkey umbenennen.

```http
DELETE /_emdash/api/auth/passkey/:id
```

Passkey löschen.

## Ratenbegrenzung

API-Endpunkte können je nach Bereitstellungskonfiguration ratenbeschränkt sein. Bei Ratenbegrenzung enthalten Antworten:

```http
HTTP/1.1 429 Too Many Requests
Retry-After: 60
```

## CORS

Die API unterstützt CORS für Browseranfragen. Konfigurieren Sie erlaubte Ursprünge in Ihrer Bereitstellung.
