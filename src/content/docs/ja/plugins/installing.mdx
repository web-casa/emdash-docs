EmDashプラグインは2つの方法でインストールできます：管理ダッシュボード経由のマーケットプレイスから、またはAstro設定に直接追加します。マーケットプレイスのプラグインは隔離されたサンドボックスで実行され、設定ベースのプラグインはインプロセスで実行されます。

## マーケットプレイスから

管理ダッシュボードには、プラグインを検索、インストール、管理できるマーケットプレイスブラウザが含まれています。

### 前提条件

マーケットプレイスのプラグインをインストールするには、サイトに以下が必要です：

1. **サンドボックスランナーの設定** — マーケットプレイスのプラグインは隔離されたV8ワーカーで実行されるため、サンドボックスランタイムが必要です：

   ```typescript title="astro.config.mjs"
   import { emdash } from "emdash/astro";

   export default defineConfig({
     integrations: [
       emdash({
         marketplace: "https://marketplace.emdashcms.com",
         sandboxRunner: true,
       }),
     ],
   });
   ```

2. **管理者アクセス** — プラグインのインストールまたは削除は管理者のみが行えます。

### 閲覧とインストール

<Steps>
1. 管理パネルを開き、**プラグイン > マーケットプレイス**に移動します
2. プラグインを閲覧または検索します
3. プラグインカードをクリックして詳細ページ（README、スクリーンショット、機能、セキュリティ監査結果）を表示します
4. **インストール**をクリックします
5. 機能同意ダイアログを確認します — これはプラグインがアクセスできるものを示します
6. インストールを確認します
</Steps>

プラグインはダウンロードされ、サイトのR2バケットに保存され、サンドボックスランナーに読み込まれます。すぐにアクティブになります。

### 機能同意

インストール前に、プラグインがアクセスを必要とするものをリストしたダイアログが表示されます：

| 機能 | 意味 |
| ---------- | ------------- |
| `read:content` | コンテンツを読み取る |
| `write:content` | コンテンツの作成、更新、削除 |
| `read:media` | メディアライブラリにアクセス |
| `write:media` | メディアのアップロードと管理 |
| `network:fetch` | 特定のホストへのネットワークリクエストを行う |

<Aside type="caution">
  信頼できる作成者のプラグインのみをインストールしてください。機能システムはサンドボックス化されたプラグインがアクセスできるものを制限しますが、`write:content`を持つプラグインはサイト上の任意のコンテンツを変更できます。
</Aside>

### セキュリティ監査

マーケットプレイスのすべてのプラグインバージョンは自動化されたセキュリティ監査を受けています。監査結果はプラグインカードに表示されます：

- **合格** — 問題なし
- **警告** — 軽微な懸念が指摘されています（調査結果を確認）
- **不合格** — 重大なセキュリティ問題が検出されました

プラグインの詳細ページでは、個々の調査結果とその重大度を含む完全な監査レポートを表示できます。

### 更新

インストール済みプラグインの新しいバージョンが利用可能な場合：

1. 管理パネルの**プラグイン**に移動します
2. マーケットプレイスのプラグインに**更新可能**バッジが表示されます
3. **更新**をクリックして変更履歴と機能の変更を確認します
4. 新しいバージョンが追加機能を必要とする場合、差分が表示され、承認が必要です
5. 更新を確認します

<Aside type="note">
  新しい機能を追加する更新には明示的な承認が必要です。以前はコンテンツを読み取るだけだったプラグインがネットワークリクエストを行いたい場合、確認前に新しい機能が強調表示されます。
</Aside>

### アンインストール

1. 管理パネルの**プラグイン**に移動します
2. 削除したいマーケットプレイスプラグインをクリックします
3. **アンインストール**をクリックします
4. プラグインの保存データを保持するか削除するかを選択します
5. 確認します

プラグインのサンドボックスコードはR2バケットから削除され、すぐに実行が停止します。

## 設定から

信頼できるプラグイン（自身のコード、またはnpm経由でインストールするパッケージ）については、Astro設定に直接追加します：

```
typescript title="astro.config.mjs"
import { defineConfig } from "astro/config";
import { emdash } from "emdash/astro";
import seoPlugin from "@emdash-cms/plugin-seo";

export default defineConfig({
  integrations: [
    emdash({
      plugins: [
        seoPlugin({ generateSitemap: true }),
      ],
    }),
  ],
});
```

設定ベースのプラグイン：

- インプロセスで実行されます（サンドボックス化されません）
- Node.js APIに完全にアクセスできます
- ビルド時およびサーバー起動時に読み込まれます
- 管理UIからインストールまたは削除できません

<Aside type="tip">
  自社コードや自身で管理するプラグインには設定ベースのプラグインを使用してください。サンドボックス隔離がセキュリティ上の利点を提供するサードパーティの拡張機能にはマーケットプレイスのプラグインを使用してください。
</Aside>

## マーケットプレイス vs. 設定：どちらを使用するか

| | マーケットプレイス | 設定 |
| --- | --- | --- |
| **実行** | サンドボックス化されたV8アイソレート | インプロセス |
| **インストール方法** | 管理UI | コード変更 + デプロイ |
| **機能** | 実行時に強制 | ドキュメントのみ |
| **Node.js API** | 利用不可 | 完全アクセス |
| **最適な用途** | サードパーティプラグイン | 自社コード |
| **更新** | 管理パネルでワンクリック | バージョンアップ + デプロイ |
