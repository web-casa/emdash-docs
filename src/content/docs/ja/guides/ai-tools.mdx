EmDashには、AIアシスタントがサイトのコンテンツを直接操作できるようにする[組み込みのMCPサーバー](https://modelcontextprotocol.io)があります。Claude、ChatGPT、またはその他のツールに、投稿の下書き作成、ページの更新、メディアの管理、コンテンツの検索などを依頼できます。すべて自然な会話を通じて行えます。

## MCPサーバーを有効にする

MCPサーバーはデフォルトで無効になっています。Astro設定で有効にしてください：

```js title="astro.config.mjs"
emdash({
  mcp: true,
})
```

## セットアップ

サイトのMCPサーバーURLは次のとおりです：

```
https://example.com/_emdash/api/mcp
```

`example.com`をあなたのドメインに置き換えてください。ローカル開発の場合は、`http://localhost:4321/_emdash/api/mcp`.`を使用してください。

<Tabs>
<TabItem label="Claude">

[claude.ai](https://claude.ai)で追加されたコネクターは、WebアプリとClaude Desktopの両方で動作します。

<Steps>

1. [設定 > コネクター](https://claude.ai/settings/connectors)に移動します

2. **カスタムコネクターを追加**をクリックします

3. サイトのMCPサーバーURLを入力します

4. **追加**をクリックします -- ブラウザが開き、ログインしてアクセスを承認します

5. 新しい会話を開始し、チャット入力欄の**+**をクリックしてから**コネクター**を選択し、サイトのトグルをオンにします

</Steps>

TeamおよびEnterpriseプランの場合、オーナーが最初に[管理者設定 > コネクター](https://claude.ai/admin-settings/connectors)からコネクターを追加します。メンバーはその後、各自の設定から個別に接続します。

</TabItem>
<TabItem label="ChatGPT">

ChatGPTは、Pro、Business、EnterpriseプランでMCPサーバーをサポートしています。

<Steps>

1. **設定 > アプリとコネクター > 詳細設定**に移動し、**開発者モード**を有効にします

2. **設定 > コネクター > 作成**に移動します

3. 名前、説明、およびサイトのMCPサーバーURLを入力します

4. **作成**をクリックします

5. 会話中、コンポーザーの近くの**+**をクリックし、次に**その他**を選択し、コネクターを選択します

</Steps>

</TabItem>

## できること

接続後、AIアシスタントに自然言語で以下の操作を依頼できます。ツール名を知る必要はありません。やりたいことを説明するだけです。

### コンテンツ

- **コンテンツを閲覧** -- 「最新の10件のブログ投稿を表示して」または「すべての下書きページを検索して」
- **コンテンツを読む** -- 「'hello-world'という投稿を取得して要約して」
- **コンテンツを作成** -- 「サマーセールについて新しいブログ投稿を書いて」または「Aboutセクションの下書きページを作成して」
- **コンテンツを編集** -- 「価格ページを新しいプランについて言及するように更新して」または「FAQ投稿のタイプミスを修正して」
- **公開とスケジュール** -- 「サマーセールの投稿を公開して」または「発表を6月1日午前9時にスケジュールして」
- **バージョンを比較** -- 「ホームページが最後に公開されてから何が変更されたか表示して」
- **下書きを管理** -- 「aboutページの下書き変更を破棄して」または「ニュースレターテンプレートを複製して」
- **翻訳** -- 「ウェルカム投稿にはどのような翻訳がありますか？」（i18nが有効な場合）

### メディア

- **メディアを閲覧** -- 「アップロードされたすべての画像を一覧表示して」または「メディアライブラリ内のPDFを表示して」
- **詳細を確認** -- 「このメディアアイテムの詳細を取得して」
- **メタデータを更新** -- 「ヒーロー画像の代替テキストを'Mountain sunset'に設定して」
- **ファイルを削除** -- 「古いバナー画像を削除して」

### 検索

- **コンテンツを検索** -- 「'accessibility'について言及している投稿を検索して」または「すべてのコレクションでTypeScriptに関するものを検索して」

### タクソノミー

- **閲覧** -- 「すべてのカテゴリを一覧表示して」または「タグを表示して」
- **用語を作成** -- 「'tutorials'タグを追加して」または「'Engineering'の下に'Frontend'サブカテゴリを作成して」

### メニュー

- **メニューを表示** -- 「メインナビゲーションメニューを表示して」または「フッターメニューには何がありますか？」

### スキーマ（管理者のみ）

- **検査** -- 「どのようなコレクションが存在しますか？」または「投稿コレクションのフィールドを表示して」
- **コレクションを作成** -- 「名前と引用フィールドを持つ新しい'testimonials'コレクションを作成して」
- **スキーマを変更** -- 「投稿に'featured'ブールフィールドを追加して」

<Aside type="caution">
	スキーマの変更（コレクションやフィールドの作成/削除）はデータベース構造を変更し、管理者権限が必要です。十分なアクセス権がない場合、AIが通知します。
</Aside>

### リビジョン

- **履歴を表示** -- 「この投稿のリビジョン履歴を表示して」
- **復元** -- 「投稿を以前のバージョンに復元して」

## 権限

AIツールを通じてできることは、EmDashでのロールによって異なります。AIアシスタントは、管理パネルで持っているのと同じ権限で動作します：

| ロール | AIができること |
| --- | --- |
| **管理者** | スキーマ変更を含むすべての操作 |
| **編集者** | すべてのコンテンツ、メディア、タクソノミー、メニュー。スキーマを閲覧可能。 |
| **投稿者** | 自身のコンテンツとメディア |
| **寄稿者** | 自身のコンテンツ（公開不可）とメディア |

アクセス権のない操作を試みた場合、AIが通知します。

## ヒント

- **コレクションについて具体的に指定する。** 複数のコレクションがある場合、「投稿を作成して」ではなく「ブログ投稿を作成して」と言います。
- **最初にスキーマを尋ねる。** コレクションにどのようなフィールドがあるかわからない場合は、コンテンツを作成または編集する前に「投稿コレクションにはどのようなフィールドがありますか？」と尋ねます。
- **公開前に確認する。** AIに下書きとしてコンテンツを作成するよう依頼し、管理パネルで確認してから、AIに公開するよう依頼するか、自分で公開します。
- **確認には比較を使用する。** 公開前に、「この投稿のライブバージョンと下書きバージョンを比較して」と依頼し、何が変更されるかを正確に確認します。
- **リッチテキストフィールドはPortable Textを使用します。** AIはリッチテキストフィールドのコンテンツを書けますが、複雑な書式設定は管理エディターで行うのが最適です。
