このガイドでは、EmDash管理ダッシュボードを使用してコンテンツを作成、編集、管理する方法について説明します。

## 管理パネルへのアクセス

サイトの `/_emdash/admin` をブラウザで開きます。セットアップ時に作成した認証情報でログインしてください。

ダッシュボードには以下が表示されます：

- **サイドバー** - コレクション、メディア、設定へのナビゲーション
- **コンテンツリスト** - 選択したコレクション内のエントリ
- **クイックアクション** - 新規コンテンツ作成、一括操作

## コンテンツの作成

1. サイドバーでコレクション名をクリックします（例：**Posts**）

2. **New Post**（または該当コレクションの同等ボタン）をクリックします

3. 必須フィールドに入力します：
   - **Title** - コンテンツの表示名
   - **Slug** - URL識別子（タイトルから自動生成、編集可能）

4. リッチテキストエディターを使用してコンテンツを追加します

5. サイドバーでメタデータを設定します：
   - **Status** - 下書き、公開、アーカイブ
   - **Publication date** - 公開日時
   - **Categories and tags** - 分類とタグの割り当て

6. **Save** をクリックします

<Aside>
	下書きは管理パネル内でのみ表示されます。コンテンツをサイト上で表示可能にするには、ステータスを **Published** に変更してください。
</Aside>

## コンテンツのステータス

各エントリは以下の3つのステータスのいずれかを持ちます：

| ステータス        | 表示範囲       | 使用例                 |
| ------------- | ---------- | ---------------- |
| **Draft**     | 管理者のみ     | 作業中             |
| **Published** | 公開         | 公開済みコンテンツ     |
| **Archived**  | 管理者のみ     | 廃止済みコンテンツ     |

ステータスはエディターサイドバーのドロップダウンで変更します。

## リッチテキストエディター

EmDashのエディターは以下をサポートしています：

- **見出し** - H2からH6
- **書式設定** - 太字、斜体、下線、取り消し線
- **リスト** - 番号付きと箇条書き
- **リンク** - 内部リンクと外部リンク
- **画像** - メディアライブラリから挿入
- **コードブロック** - シンタックスハイライト付き
- **埋め込み** - YouTube、Vimeo、Twitter
- **セクション** - `/section` コマンドによる再利用可能なコンテンツブロック

### スラッシュコマンド

`/` を入力してクイック挿入コマンドにアクセスします：

| コマンド                      | アクション                              |
| ---------------------------- | ----------------------------------- |
| `/section`                   | 再利用可能なセクションを挿入           |
| `/image`                     | メディアライブラリから画像を挿入        |
| `/code`                      | コードブロックを挿入                    |

### キーボードショートカット

| アクション | ショートカット               |
| ------ | ---------------------- |
| 太字   | `Ctrl/Cmd + B`         |
| 斜体   | `Ctrl/Cmd + I`         |
| リンク | `Ctrl/Cmd + K`         |
| 元に戻す | `Ctrl/Cmd + Z`         |
| やり直し | `Ctrl/Cmd + Shift + Z` |
| 保存   | `Ctrl/Cmd + S`         |

### 画像の挿入

1. ツールバーの画像ボタンをクリックします

2. メディアライブラリから既存の画像を選択するか、新しい画像をアップロードします

3. 代替テキストを追加します（アクセシビリティのため必須）

4. 配置とサイズのオプションを調整します

5. **Insert** をクリックします

## コンテンツの編集

1. コンテンツを含むコレクションに移動します

2. 編集したいエントリをクリックします

3. 変更を加えます

4. **Save** をクリックします

公開済みコンテンツへの変更は、サイト上に即時反映されます。再ビルドは不要です。

### リビジョン履歴

EmDashはコンテンツへの変更を追跡します。エディターサイドバーからリビジョン履歴にアクセスします：

1. エディターサイドバーの **Revisions** をクリックします

2. タイムスタンプ付きの以前のバージョンのリストを表示します

3. リビジョンをクリックしてプレビューします

4. **Restore** をクリックしてそのバージョンに戻します

<Aside type="caution">
	リビジョンを復元すると、復元されたコンテンツで新しいリビジョンが作成されます。元のリビジョン履歴は保持されます。
</Aside>

## 一括操作

複数のエントリに対して一度にアクションを実行します：

1. コンテンツリストでチェックボックスを使用してエントリを選択します

2. **Bulk Actions** ドロップダウンをクリックします

3. アクションを選択します：
   - **Publish** - 選択したすべてを公開済みに設定
   - **Archive** - 選択したすべてをアーカイブ済みに設定
   - **Delete** - 選択したものを完全に削除

4. アクションを確認します

## 検索とフィルタリング

### 検索

検索ボックスを使用して、タイトルまたはコンテンツでコンテンツを検索します。検索は大文字小文字を区別せず、部分一致します。

### フィルター

コンテンツリストを以下でフィルタリングします：

- **Status** - 下書き、公開済み、アーカイブ済み
- **Date range** - 作成日または更新日
- **Author** - コンテンツ作成者
- **Taxonomy** - カテゴリーまたはタグの割り当て

**Clear Filters** をクリックしてリセットします。

## コンテンツのスケジューリング

コンテンツを将来の日時に公開するようにスケジュールします：

1. コンテンツを作成または編集します

2. ステータスを **Draft** に設定します

3. **Publication date** を将来の日時に設定します

4. **Save** をクリックします

公開日時が到来すると、コンテンツは自動的に公開済みになります。

<Aside type="tip">
	スケジュールされた公開を機能させるには、サイトがサーバーサイドレンダリングされているか、保留中の公開をチェックするスケジュールタスクが必要です。
</Aside>

## コンテンツの削除

編集画面またはコンテンツリストからコンテンツを削除します：

### エディターから

1. 削除したいコンテンツを開きます

2. ツールバーの **Delete** をクリックします

3. 削除を確認します

### リストから

1. チェックボックスを使用してエントリを選択します

2. **Bulk Actions** > **Delete** をクリックします

3. 削除を確認します

<Aside type="caution">
	削除されたコンテンツは完全に削除され、復元できません。後で必要になる可能性がある場合は、代わりにアーカイブすることを検討してください。
</Aside>

## コンテンツAPI

プログラムによるアクセスのためには、EmDash管理APIを使用します：

### コンテンツの作成

```bash
POST /_emdash/api/content/posts
Content-Type: application/json
Authorization: Bearer YOUR_API_TOKEN

{
  "title": "My New Post",
  "slug": "my-new-post",
  "content": "<p>Post content here</p>
",
  "status": "draft"
}
```

### コンテンツの更新

```bash
PUT /_emdash/api/content/posts/my-new-post
Content-Type: application/json
Authorization: Bearer YOUR_API_TOKEN

{
  "title": "Updated Title",
  "status": "published"
}
```

### コンテンツの削除

```bash
DELETE /_emdash/api/content/posts/my-new-post
Authorization: Bearer YOUR_API_TOKEN
```

## コンテンツの翻訳

[i18nが有効になっている](/ja/guides/internationalization/)場合、任意のコンテンツエントリの翻訳を作成できます。

### 翻訳の作成

1. 翻訳したいコンテンツエントリを開きます

2. エディターサイドバーで **Translations** パネルを見つけます

3. ターゲットロケールの横にある **Translate** をクリックします

4. 事前入力されたコンテンツを編集します — 新しい言語用にタイトル、スラッグ、本文を調整します

5. **Save** をクリックします

新しい翻訳は元のエントリにリンクされ、下書きとして開始されます。翻訳が準備できたら独立して公開してください。

### 翻訳間の切り替え

Translationsパネルには設定されたすべてのロケールが表示されます。既存の翻訳の横にある **Edit** をクリックすると、直接その翻訳に移動できます。現在のロケールはチェックマークでマークされます。

### ロケールフィルター

コンテンツリストでは、ツールバーのロケールドロップダウンを使用して、言語別にエントリーをフィルタリングできます。各エントリーには専用の列でロケールが表示されます。

<Aside type="tip">
  各翻訳には独自のスラッグ、ステータス、改訂履歴があります。翻訳を個別に公開、スケジュール、管理できます。
</Aside>

設定、クエリ、言語切り替え機能の詳細については、[国際化ガイド](/ja/guides/internationalization/)をご覧ください。
