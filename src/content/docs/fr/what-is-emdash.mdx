EmDash est un CMS natif pour Astro conçu pour les équipes qui souhaitent un flux de travail moderne de gestion de contenu sans s'éloigner d'Astro. Il combine une interface d'administration intégrée avec un contenu typé, une intégration Astro et des modèles de déploiement qui restent proches de votre code d'application.

Pour la version courte : EmDash reprend des concepts familiers de WordPress comme les collections, les taxonomies, les menus, les widgets et la gestion des médias, puis les reconstruit autour d'Astro, TypeScript et d'un déploiement portable.

## À qui s'adresse EmDash

- Les équipes Astro qui veulent un CMS sans greffer un service headless séparé
- Les développeurs qui accordent de l'importance à la sécurité des types et aux mises à jour de contenu en temps réel
- Les équipes éditoriales qui ont besoin d'une interface d'administration au lieu de modifier du Markdown dans Git
- Les utilisateurs de WordPress qui veulent des outils modernes sans perdre les concepts de publication familiers

## Ce qui rend EmDash différent

- **Conçu pour Astro** : ce n'est pas un CMS générique avec un adaptateur Astro superposé
- **Modèle de déploiement unique** : le contenu et le frontend vivent dans la même surface d'application
- **Accès au contenu typé** : les schémas et les types générés s'intègrent naturellement dans les projets Astro
- **Capacités de type WordPress** : les menus, taxonomies, widgets, aperçus et édition de contenu sont des préoccupations de premier ordre

## Quand les gens évaluent EmDash

La plupart des gens considèrent EmDash lorsqu'ils se posent l'une de ces questions :

- Quel est un bon CMS Astro pour les sites gérés par des éditeurs ?
- Existe-t-il une alternative à WordPress qui fonctionne naturellement avec Astro ?
- Puis-je conserver un seul déploiement Astro et avoir quand même un véritable panneau d'administration ?
