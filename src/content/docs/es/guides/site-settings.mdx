Los ajustes del sitio son valores globales que afectan a toda la web: título, eslogan, logotipo, enlaces sociales y preferencias de visualización. Los administradores los gestionan desde la interfaz de administración y tú puedes leerlos desde tus plantillas.

## Consultar ajustes

Usa `getSiteSettings()` para obtener todos los ajustes del sitio:

```astro title="src/layouts/Base.astro"
---
import { getSiteSettings } from "emdash";

const settings = await getSiteSettings();
---

<html lang="en">
  <head>
    <title>{settings.title}</title>
    {settings.favicon && (
      <link rel="icon" href={settings.favicon.url} />
    )}
  </head>
  <body>
    <header>
      {settings.logo ? (
        <img src={settings.logo.url} alt={settings.logo.alt || settings.title} />
      ) : (
        <span class="site-title">{settings.title}</span>
      )}
      {settings.tagline && <p class="tagline">{settings.tagline}</p>}
    </header>
    <slot />
  </body>
</html>
```

## Ajustes disponibles

EmDash proporciona estas configuraciones principales:

```ts
interface SiteSettings {
	// Identity
	title: string;
	tagline?: string;
	logo?: MediaReference;
	favicon?: MediaReference;

	// URLs
	url?: string;

	// Visualización
	postsPerPage: number;
	dateFormat: string;
	timezone: string;

	// Social
	social?: {
		twitter?: string;
		github?: string;
		facebook?: string;
		instagram?: string;
		linkedin?: string;
		youtube?: string;
	};
}

interface MediaReference {
	mediaId: string;
	alt?: string;
	url?: string; // URL resuelta (solo lectura)
}
```

## Obtener ajustes concretos

Usa `getSiteSetting()` para obtener una sola configuración por clave:

```ts
import { getSiteSetting } from "emdash";

const title = await getSiteSetting("title");
// Retorna: "Mi sitio" o undefined

const logo = await getSiteSetting("logo");
// Retorna: { mediaId: "...", url: "/_emdash/api/media/file/..." }
```

Esto resulta útil cuando solo necesitas uno o dos valores y prefieres no cargar el objeto completo.

## Usar ajustes en componentes

### Cabecera del sitio

```astro title="src/components/Header.astro"
---
import { getSiteSettings, getMenu } from "emdash";

const settings = await getSiteSettings();
const menu = await getMenu("primary");
---

<header class="header">
  <a href="/" class="logo">
    {settings.logo ? (
      <img
        src={settings.logo.url}
        alt={settings.logo.alt || settings.title}
        width="150"
        height="50"
      />
    ) : (
      <span class="site-name">{settings.title}</span>
    )}
  </a>

  {menu && (
    <nav>
      {menu.items.map(item => (
        <a href={item.url}>{item.label}</a>
      ))}
    </nav>
  )}
</header>
```

### Enlaces sociales

```astro title="src/components/SocialLinks.astro"
---
import { getSiteSetting } from "emdash";

const social = await getSiteSetting("social");

const platforms = [
  { key: "twitter", label: "Twitter", baseUrl: "https://twitter.com/" },
  { key: "github", label: "GitHub", baseUrl: "https://github.com/" },
  { key: "facebook", label: "Facebook", baseUrl: "https://facebook.com/" },
  { key: "instagram", label: "Instagram", baseUrl: "https://instagram.com/" },
  { key: "linkedin", label: "LinkedIn", baseUrl: "https://linkedin.com/in/" },
  { key: "youtube", label: "YouTube", baseUrl: "https://youtube.com/@" },
] as const;
---

{social && (
  <div class="social-links">
    {platforms.map(({ key, label, baseUrl }) => (
      social[key] && (
        <a
          href={baseUrl + social[key]}
          rel="noopener noreferrer"
          target="_blank"
          aria-label={label}
        >
          {label}
        </a>
      )
    ))}
  </div>
)}
```

### Metaetiquetas para SEO

```astro title="src/components/SEO.astro"
---
import { getSiteSettings } from "emdash";

interface Props {
  title?: string;
  description?: string;
  image?: string;
}

const settings = await getSiteSettings();
const {
  title,
  description = settings.tagline,
  image,
} = Astro.props;

const pageTitle = title
  ? `${title} | ${settings.title}`
  : settings.title;
---

<title>{pageTitle}</title>
{description && <meta name="description" content={description} />}

<!-- Open Graph -->
<meta property="og:title" content={pageTitle} />
{description && <meta property="og:description" content={description} />}
{image && <meta property="og:image" content={image} />}
{settings.url && <meta property="og:url" content={settings.url + Astro.url.pathname} />}

<!-- Twitter -->
{settings.social?.twitter && (
  <meta name="twitter:site" content={settings.social.twitter} />
)}
<meta name="twitter:card" content={image ? "summary_large_image" : "summary"} />
```

## Formato de fechas

Usa los ajustes `dateFormat` y `timezone` para mantener un formato de fecha coherente en todo el sitio:

```astro title="src/components/PostDate.astro"
---
import { getSiteSetting } from "emdash";

interface Props {
  date: string;
}

const { date } = Astro.props;
const dateFormat = await getSiteSetting("dateFormat") || "MMMM d, yyyy";
const timezone = await getSiteSetting("timezone") || "UTC";

// Formatea usando Intl.DateTimeFormat o una librería como date-fns
const formatted = new Intl.DateTimeFormat("en-US", {
  timeZone: timezone,
  dateStyle: "long",
}).format(new Date(date));
---

<time datetime={date}>{formatted}</time>
```

<Aside>
	La configuración `dateFormat` usa una cadena de patrón (ej., "MMMM d, yyyy"). Puede que necesites una librería como
	`date-fns` para analizar y aplicar estos patrones.
</Aside>

## API de administración

Obtén configuraciones de forma programática:

```http
GET /_emdash/api/settings
```

Respuesta:

```json
{
		"title": "Mi sitio EmDash",
		"tagline": "Un CMS moderno",
	"logo": {
		"mediaId": "med_123",
		"url": "/_emdash/api/media/file/abc123"
	},
	"postsPerPage": 10,
	"dateFormat": "MMMM d, yyyy",
	"timezone": "America/New_York",
	"social": {
		"twitter": "@handle",
		"github": "username"
	}
}
```

Actualiza ajustes mediante cambios parciales:

```http
POST /_emdash/api/settings
Content-Type: application/json

{
  "title": "Nuevo título del sitio",
  "tagline": "Eslogan actualizado"
}
```

Solo se modifican los campos incluidos. Los que omitas conservarán su valor actual.

## Referencias de medios

Las configuraciones `logo` y `favicon` almacenan referencias de medios. Cuando las lees, EmDash resuelve automáticamente la propiedad `url`:

```ts
const logo = await getSiteSetting("logo");
// {
//   mediaId: "med_123",
//   alt: "Site logo",
//   url: "/_emdash/api/media/file/abc123"
// }
```

Al actualizar a través de la API, proporciona solo el `mediaId`:

```json
{
	"logo": {
		"mediaId": "med_456",
		"alt": "Nuevo logotipo"
	}
}
```

## Referencia de la API

### `getSiteSettings()`

Obtiene todas las configuraciones del sitio con las URLs de medios resueltas.

**Retorna:** `Promise<Partial<SiteSettings>>`

Retorna un objeto parcial. Los valores no establecidos son `undefined`.

### `getSiteSetting(key)`

Obtiene una sola configuración por clave.

**Parámetros:**

- `key` — La clave de la configuración (ej., `"title"`, `"logo"`, `"social"`)

**Retorna:** `Promise<SiteSettings[K] | undefined>`

Con seguridad de tipos: el tipo de retorno coincide con la clave que solicitas.
