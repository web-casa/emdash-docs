이 가이드는 EmDash 관리자 대시보드를 사용하여 콘텐츠를 생성, 편집 및 관리하는 방법을 다룹니다.

## 관리자 접근

사이트의 `/_emdash/admin` 주소로 브라우저를 열어 접속하세요. 설정 과정에서 생성한 자격 증명으로 로그인하세요.

대시보드에는 다음이 표시됩니다:

- **사이드바** - 콜렉션, 미디어, 설정으로의 탐색
- **콘텐츠 목록** - 선택된 콜렉션의 항목들
- **빠른 작업** - 새 콘텐츠 생성, 대량 작업

## 콘텐츠 생성

1. 사이드바에서 콜렉션 이름을 클릭하세요 (예: **Posts**).

2. **New Post** (또는 해당 콜렉션에 맞는 항목)을 클릭하세요.

3. 필수 필드를 작성하세요:
   - **Title** - 콘텐츠의 표시 이름
   - **Slug** - URL 식별자 (제목에서 자동 생성되며 편집 가능)

4. 리치 텍스트 에디터를 사용하여 콘텐츠를 추가하세요.

5. 사이드바에서 메타데이터를 설정하세요:
   - **Status** - 초안, 게시됨 또는 보관됨
   - **Publication date** - 게시할 시점
   - **Categories and tags** - 분류 체계 할당

6. **Save**를 클릭하세요.

<Aside>
	초안은 관리자에서만 볼 수 있습니다. 콘텐츠를 사이트에 공개하려면 상태를 **Published**로 변경하세요.
</Aside>

## 콘텐츠 상태

모든 항목은 세 가지 상태 중 하나를 가집니다:

| 상태           | 가시성       | 사용 사례               |
| -------------- | ------------ | ----------------------- |
| **Draft**      | 관리자 전용  | 작업 중인 내용          |
| **Published**  | 공개         | 라이브 콘텐츠           |
| **Archived**   | 관리자 전용  | 사용 중단된 콘텐츠      |

에디터 사이드바의 드롭다운을 사용하여 상태를 변경하세요.

## 리치 텍스트 에디터

EmDash의 에디터는 다음을 지원합니다:

- **Headings** - H2부터 H6까지
- **Formatting** - 굵게, 기울임꼴, 밑줄, 취소선
- **Lists** - 순서 있는 목록과 순서 없는 목록
- **Links** - 내부 및 외부 링크
- **Images** - 미디어 라이브러리에서 삽입
- **Code blocks** - 구문 강조 기능 포함
- **Embeds** - YouTube, Vimeo, Twitter
- **Sections** - `/section` 명령어를 통한 재사용 가능한 콘텐츠 블록

### 슬래시 명령어

빠른 삽입 명령어에 접근하려면 `/`를 입력하세요:

| 명령어                       | 동작                                  |
| ---------------------------- | ------------------------------------- |
| `/section`                   | 재사용 가능한 섹션 삽입               |
| `/image`                     | 미디어 라이브러리에서 이미지 삽입     |
| `/code`                      | 코드 블록 삽입                        |

### 키보드 단축키

| 동작     | 단축키                     |
| -------- | -------------------------- |
| Bold     | `Ctrl/Cmd + B`             |
| Italic   | `Ctrl/Cmd + I`             |
| Link     | `Ctrl/Cmd + K`             |
| Undo     | `Ctrl/Cmd + Z`             |
| Redo     | `Ctrl/Cmd + Shift + Z`     |
| Save     | `Ctrl/Cmd + S`             |

### 이미지 삽입

1. 도구 모음의 이미지 버튼을 클릭하세요.

2. 미디어 라이브러리에서 기존 이미지를 선택하거나 새 이미지를 업로드하세요.

3. 대체 텍스트를 추가하세요 (접근성을 위해 필수).

4. 정렬 및 크기 옵션을 조정하세요.

5. **Insert**를 클릭하세요.

## 콘텐츠 편집

1. 해당 콘텐츠가 포함된 콜렉션으로 이동하세요.

2. 편집하려는 항목을 클릭하세요.

3. 변경 사항을 적용하세요.

4. **Save**를 클릭하세요.

게시된 콘텐츠에 대한 변경 사항은 사이트에 즉시 반영됩니다. 재빌드가 필요하지 않습니다.

### 수정 기록

EmDash는 콘텐츠 변경 사항을 추적합니다. 에디터 사이드바에서 수정 기록에 접근하세요:

1. 에디터 사이드바에서 **Revisions**를 클릭하세요.

2. 타임스탬프가 있는 이전 버전 목록을 확인하세요.

3. 버전을 클릭하여 미리 보세요.

4. **Restore**를 클릭하여 해당 버전으로 되돌리세요.

<Aside type="caution">
	버전을 복원하면 복원된 콘텐츠로 새 수정 기록이 생성됩니다. 원본 수정 기록은 보존됩니다.
</Aside>

## 대량 작업

여러 항목에 대해 한 번에 작업을 수행하세요:

1. 콘텐츠 목록에서 체크박스를 사용하여 항목을 선택하세요.

2. **Bulk Actions** 드롭다운을 클릭하세요.

3. 작업을 선택하세요:
   - **Publish** - 선택된 모든 항목을 게시됨으로 설정
   - **Archive** - 선택된 모든 항목을 보관됨으로 설정
   - **Delete** - 선택된 항목을 영구적으로 제거

4. 작업을 확인하세요.

## 검색 및 필터링

### 검색

검색 상자를 사용하여 제목이나 콘텐츠로 콘텐츠를 찾으세요. 검색은 대소문자를 구분하지 않으며 부분 단어와 일치합니다.

### 필터

다음 기준으로 콘텐츠 목록을 필터링하세요:

- **Status** - 초안, 게시됨, 보관됨
- **Date range** - 생성 또는 수정 날짜 범위
- **Author** - 콘텐츠 생성자
- **Taxonomy** - 카테고리 또는 태그 할당

**Clear Filters**를 클릭하여 초기화하세요.

## 콘텐츠 예약

콘텐츠를 미래 날짜에 게시하도록 예약하세요:

1. 콘텐츠를 생성하거나 편집하세요.

2. 상태를 **Draft**로 설정하세요.

3. **Publication date**를 미래 날짜와 시간으로 설정하세요.

4. **Save**를 클릭하세요.

게시 날짜가 도래하면 콘텐츠가 자동으로 게시됨 상태가 됩니다.

<Aside type="tip">
	예약된 게시는 사이트가 서버 렌더링되거나 보류 중인 게시물을 확인하는 예약 작업이 있어야 합니다.
</Aside>

## 콘텐츠 삭제

편집 화면이나 콘텐츠 목록에서 콘텐츠를 삭제하세요:

### 에디터에서

1. 삭제하려는 콘텐츠를 열세요.

2. 도구 모음에서 **Delete**를 클릭하세요.

3. 삭제를 확인하세요.

### 목록에서

1. 체크박스를 사용하여 항목을 선택하세요.

2. **Bulk Actions** > **Delete**를 클릭하세요.

3. 삭제를 확인하세요.

<Aside type="caution">
	삭제된 콘텐츠는 영구적으로 제거되며 복구할 수 없습니다. 나중에 콘텐츠가 필요할 수 있다면 대신 보관하는 것을 고려하세요.
</Aside>

## 콘텐츠 API

프로그래밍 방식으로 접근하려면 EmDash 관리자 API를 사용하세요:

### 콘텐츠 생성

```bash
POST /_emdash/api/content/posts
Content-Type: application/json
Authorization: Bearer YOUR_API_TOKEN

{
  "title": "My New Post",
  "slug": "my-new-post",
  "content": "<p>Post content here</p>
",
  "status": "draft"
}
```

### 콘텐츠 업데이트

```bash
PUT /_emdash/api/content/posts/my-new-post
Content-Type: application/json
Authorization: Bearer YOUR_API_TOKEN

{
  "title": "Updated Title",
  "status": "published"
}
```

### 콘텐츠 삭제

```bash
DELETE /_emdash/api/content/posts/my-new-post
Authorization: Bearer YOUR_API_TOKEN
```

## 콘텐츠 번역

[i18n이 활성화된 경우](/ko/guides/internationalization/), 모든 콘텐츠 항목의 번역을 생성할 수 있습니다.

### 번역 생성

1. 번역하려는 콘텐츠 항목을 열세요.

2. 에디터 사이드바에서 **Translations** 패널을 찾으세요.

3. 대상 로케일 옆의 **Translate**를 클릭하세요.

4. 미리 채워진 콘텐츠를 편집하세요 — 새 언어에 맞게 제목, 슬러그 및 본문을 조정하세요.

5. **Save**를 클릭하세요.

새 번역은 원본 항목에 연결되며 초안 상태로 시작합니다. 번역이 준비되면 독립적으로 게시하세요.

### 번역 간 전환

Translations 패널에는 구성된 모든 로케일이 표시됩니다. 기존 번역 옆의 **Edit**를 클릭하여 직접 해당 번역으로 이동하세요. 현재 로케일은 체크 표시로 표시됩니다.

### 로케일 필터

콘텐츠 목록에서 툴바의 로케일 드롭다운을 사용하여 언어별로 항목을 필터링할 수 있습니다. 각 항목은 전용 열에 로케일을 표시합니다.

<Aside type="tip">
  각 번역은 고유한 슬러그, 상태 및 수정 기록을 가집니다. 번역을 독립적으로 게시, 예약 및 관리하세요.
</Aside>

구성, 쿼리 및 언어 전환기에 대한 전체 세부 정보는 [국제화 가이드](/ko/guides/internationalization/)를 참조하세요.
