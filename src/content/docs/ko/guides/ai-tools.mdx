EmDash는 AI 어시스턴트가 사이트 콘텐츠와 직접 작업할 수 있도록 내장된 [MCP 서버](https://modelcontextprotocol.io)를 갖추고 있습니다. Claude, ChatGPT 또는 기타 도구에 게시물 초안 작성, 페이지 업데이트, 미디어 관리, 콘텐츠 검색 등을 요청할 수 있습니다 — 모두 자연스러운 대화를 통해 가능합니다.

## MCP 서버 활성화

MCP 서버는 기본적으로 비활성화되어 있습니다. Astro 설정에서 활성화하세요:

```js title="astro.config.mjs"
emdash({
  mcp: true,
})
```

## 설정하기

사이트의 MCP 서버 URL은 다음과 같습니다:

```
https://example.com/_emdash/api/mcp
```

`example.com`을 자신의 도메인으로 바꾸세요. 로컬 개발의 경우 `http://localhost:4321/_emdash/api/mcp`.`을 사용하세요.

<Tabs>
<TabItem label="Claude">

[claude.ai](https://claude.ai)에서 추가한 커넥터는 웹 앱과 Claude 데스크톱 모두에서 작동합니다.

<Steps>

1. [설정 > 커넥터](https://claude.ai/settings/connectors)로 이동하세요.

2. **사용자 지정 커넥터 추가**를 클릭하세요.

3. 사이트의 MCP 서버 URL을 입력하세요.

4. **추가**를 클릭하세요 — 브라우저가 열려 로그인하고 액세스 권한을 승인하게 됩니다.

5. 새 대화를 시작하고, 채팅 입력란의 **+**를 클릭한 다음 **커넥터**를 선택하고 사이트를 토글하여 켜세요.

</Steps>

팀 및 엔터프라이즈 플랜의 경우, 소유자가 먼저 [관리자 설정 > 커넥터](https://claude.ai/admin-settings/connectors)에서 커넥터를 추가합니다. 구성원은 각자의 설정에서 개별적으로 연결합니다.

</TabItem>
<TabItem label="ChatGPT">

ChatGPT는 Pro, Business 및 Enterprise 플랜에서 MCP 서버를 지원합니다.

<Steps>

1. **설정 > 앱 및 커넥터 > 고급 설정**으로 이동하여 **개발자 모드**를 활성화하세요.

2. **설정 > 커넥터 > 만들기**로 이동하세요.

3. 이름, 설명 및 사이트의 MCP 서버 URL을 입력하세요.

4. **만들기**를 클릭하세요.

5. 대화 중에 작성기 근처의 **+**를 클릭한 다음 **더 보기**를 선택하고 커넥터를 선택하세요.

</Steps>

</TabItem>
</Tabs>

## 가능한 작업

연결되면, AI 어시스턴트에게 자연어로 이러한 작업 중 어떤 것이든 수행하도록 요청할 수 있습니다. 도구 이름을 알 필요가 없습니다 — 원하는 것을 설명하기만 하면 됩니다.

### 콘텐츠

- **콘텐츠 탐색** — "최신 블로그 게시물 10개 보여줘" 또는 "모든 초안 페이지 찾아줘"
- **콘텐츠 읽기** — "hello-world라는 게시물 가져와서 요약해줘"
- **콘텐츠 생성** — "여름 세일에 관한 새 블로그 게시물 작성해줘" 또는 "About 섹션용 초안 페이지 만들어줘"
- **콘텐츠 편집** — "가격 페이지를 새 플랜을 언급하도록 업데이트해줘" 또는 "FAQ 게시물의 오타 수정해줘"
- **게시 및 예약** — "여름 세일 게시물 게시해줘" 또는 "발표를 6월 1일 오전 9시로 예약해줘"
- **버전 비교** — "홈페이지가 마지막으로 게시된 이후 무엇이 변경되었는지 보여줘"
- **초안 관리** — "about 페이지의 초안 변경사항을 버려줘" 또는 "뉴스레터 템플릿 복제해줘"
- **번역** — "환영 게시물에 어떤 번역이 존재하나요?" (i18n이 활성화된 경우)

### 미디어

- **미디어 탐색** — "업로드된 모든 이미지 목록 보여줘" 또는 "미디어 라이브러리의 PDF 파일 보여줘"
- **세부 정보 확인** — "이 미디어 항목의 세부 정보 가져와줘"
- **메타데이터 업데이트** — "히어로 이미지의 대체 텍스트를 'Mountain sunset'으로 설정해줘"
- **파일 제거** — "오래된 배너 이미지 삭제해줘"

### 검색

- **콘텐츠 찾기** — "'접근성'을 언급한 게시물 검색해줘" 또는 "모든 컬렉션에서 TypeScript에 관한 내용 찾아줘"

### 분류 체계

- **탐색** — "모든 카테고리 목록 보여줘" 또는 "태그 보여줘"
- **용어 생성** — "'tutorials' 태그 추가해줘" 또는 "'Engineering' 아래에 'Frontend' 하위 카테고리 생성해줘"

### 메뉴

- **메뉴 보기** — "주 네비게이션 메뉴 보여줘" 또는 "푸터 메뉴에 뭐가 있나요?"

### 스키마 (관리자 전용)

- **검사** — "어떤 컬렉션이 존재하나요?" 또는 "posts 컬렉션의 필드 보여줘"
- **컬렉션 생성** — "name 및 quote 필드를 가진 새 'testimonials' 컬렉션 생성해줘"
- **스키마 수정** — "posts에 'featured' 불리언 필드 추가해줘"

<Aside type="caution">
	스키마 변경(컬렉션 및 필드 생성/삭제)은 데이터베이스 구조를 수정하며 관리자 권한이 필요합니다. 권한이 부족한 경우 AI가 알려줍니다.
</Aside>

### 개정

- **기록 보기** — "이 게시물의 개정 기록 보여줘"
- **복원** — "게시물을 이전 버전으로 복원해줘"

## 권한

AI 도구를 통해 할 수 있는 작업은 EmDash 역할에 따라 다릅니다. AI 어시스턴트는 관리자 패널에서 가지고 있는 것과 동일한 권한으로 작동합니다:

| 역할 | AI가 할 수 있는 작업 |
| --- | --- |
| **관리자** | 스키마 변경을 포함한 모든 작업 |
| **편집자** | 모든 콘텐츠, 미디어, 분류 체계 및 메뉴. 스키마를 볼 수 있음. |
| **작성자** | 자신의 콘텐츠와 미디어 |
| **기여자** | 자신의 콘텐츠(게시 불가)와 미디어 |

액세스 권한이 없는 작업을 시도하면 AI가 알려줍니다.

## 팁

- **컬렉션에 대해 구체적으로 말하세요.** 여러 컬렉션이 있는 경우 "게시물 생성"보다 "블로그 게시물 생성"이라고 말하세요.
- **먼저 스키마를 요청하세요.** 컬렉션에 어떤 필드가 있는지 확실하지 않다면, 콘텐츠를 생성하거나 편집하기 전에 "posts 컬렉션에는 어떤 필드가 있나요?"라고 물어보세요.
- **게시 전에 검토하세요.** AI에게 콘텐츠를 초안으로 생성하도록 요청하고, 관리자 패널에서 검토한 다음, AI에게 게시하도록 요청하거나 직접 게시하세요.
- **검토에는 비교 기능을 사용하세요.** 게시하기 전에 "이 게시물의 라이브 버전과 초안 버전 비교해줘"라고 요청하여 정확히 무엇이 변경될지 확인하세요.
- **리치 텍스트 필드는 Portable Text를 사용합니다.** AI는 리치 텍스트 필드용 콘텐츠를 작성할 수 있지만, 복잡한 서식은 관리자 편집기에서 수행하는 것이 가장 좋습니다.

## 개발자를 위해

MCP 서버 엔드포인트, 인증 방법, OAuth 검색, 도구 매개변수 및 오류 처리에 대한 내용은 [MCP 서버 참조](/ko/reference/mcp-server)에 문서화되어 있습니다.
