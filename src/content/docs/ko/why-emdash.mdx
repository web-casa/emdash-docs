EmDash는 전통적인 CMS 패턴과 현대 웹 개발을 결합한 Astro 네이티브 CMS입니다: 콘텐츠 편집 인터페이스, Astro 프레임워크 통합, 유연한 배포 옵션을 제공합니다.

## EmDash의 차별점

### Astro 네이티브 아키텍처

EmDash는 범용 CMS에서 개조된 것이 아니라 Astro를 위해 특별히 제작되었습니다. 콘텐츠는 Astro의 Live Content Collections를 통해 쿼리되며, 사이트와 동일한 배포 환경에 존재합니다. 별도의 서비스, API 왕복, 웹훅 동기화가 필요 없습니다.

### 익숙한 콘텐츠 모델

WordPress를 사용해 본 적이 있다면, EmDash의 컬렉션(게시물 유형과 유사), 분류, 메뉴, 위젯 영역, 미디어 라이브러리와 같은 개념이 익숙하게 느껴질 것입니다. 정신적 모델은 이전되지만, 구현은 현대적인 도구를 사용합니다.

### 프레임워크 통합

EmDash는 Astro를 위해 목적에 맞게 제작되었습니다. 이 긴밀한 통합은 타입 안전 쿼리, 컴포넌트 수준 캐싱, 통합된 미리보기를 가능하게 합니다.

## 다른 CMS와의 비교

다른 CMS 접근 방식은 서로 다른 요구 사항에 적합합니다:

| 측면              | 전통적 CMS  | 헤드리스 CMS    | EmDash              |
| ------------------- | ---------------- | --------------- | --------------------- |
| **아키텍처**    | 모놀리식       | 분리형       | Astro와 통합됨 |
| **콘텐츠 편집** | 내장된 관리자   | 내장된 관리자  | 내장된 관리자        |
| **프론트엔드**        | 테마/템플릿 | 직접 가져오기  | Astro 컴포넌트      |
| **배포**      | 단일 서버    | CMS + 프론트엔드  | 단일 배포     |
| **타입 안전성**     | 런타임          | API 타입       | 완전한 TypeScript       |
| **콘텐츠 업데이트** | 즉시적        | 웹훅/재빌드 | 즉시적 (SSR)       |
| **플러그인 모델**    | 동일 프로세스     | API 확장  | 후크를 통한 샌드박스화  |

## Cloudflare 배포

EmDash는 SQLite와 S3 호환 스토리지가 있는 모든 플랫폼에서 실행됩니다. 또한 Cloudflare 특정 기능을 지원합니다:

- **D1** — 자동 복제 기능이 있는 에지의 SQLite
- **R2** — 이그레스 비용이 없는 S3 호환 스토리지
- **Workers** — 빠른 콜드 스타트를 가진 글로벌 배포

## 플러그인 마이그레이션

EmDash는 WordPress 플러그인 기능을 마이그레이션하는 데 도움이 되는 도구를 제공합니다:

- **개념 매핑** — WordPress 후크, 필터 및 API를 EmDash의 동등한 기능에 매핑
- **마이그레이션 가이드** — 특정 플러그인 패턴을 이식하기 위한 문서
- **AI 지원 이식** — AI 도구가 WordPress 플러그인 코드에서 EmDash 플러그인을 생성하도록 돕는 구조화된 문서

복잡한 플러그인은 여전히 사람의 검토가 필요하지만, 간단한 플러그인의 경우 마이그레이션 가이드가 이식 작업을 줄여줍니다.

## EmDash 사용 시기

**EmDash는 다음과 같은 경우를 위해 설계되었습니다:**

- CMS가 필요한 새로운 Astro 프로젝트
- 현대적인 도구를 원하는 WordPress 마이그레이션
- 콘텐츠 편집자가 코드를 건드리지 않아야 하는 사이트
- Cloudflare에 배포하는 프로젝트
- 타입 안전성과 개발자 경험이 중요한 사이트

**EmDash는 다음과 같은 경우 적합하지 않을 수 있습니다:**

- Astro가 아닌 프로젝트 (Astro와 긴밀하게 결합되어 있음)
- 전자상거래 (WooCommerce 규모의 기능은 아직 사용 불가)
- 만족하는 기존 헤드리스 아키텍처
- WordPress의 특정 플러그인 생태계가 필요한 프로젝트
